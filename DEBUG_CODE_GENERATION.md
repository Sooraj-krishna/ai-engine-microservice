# Code Generation Debug Mode

## Overview

This feature allows you to see the raw code generated by Gemini **BEFORE** any validation or processing happens. This is useful for debugging code generation failures.

## How It Works

When enabled, the system will save:

1. **Raw Generated Code** - The exact code returned by Gemini in `data/raw_generated_code/`
2. **Validation Results** - Detailed validation reports in `data/validation_results/`

## Quick Start

### Enable Debug Mode (Default: ON)

Debug mode is **enabled by default**. The following environment variables control it:

```bash
# Already set to 'true' by default
export SAVE_RAW_GENERATED_CODE=true   # Saves raw Gemini responses
export SAVE_VALIDATION_DEBUG=true      # Saves validation results
```

### Disable Debug Mode

If you want to disable debug logging:

```bash
export SAVE_RAW_GENERATED_CODE=false
export SAVE_VALIDATION_DEBUG=false
```

## Where to Find Debug Files

### 1. Raw Generated Code

**Location**: `data/raw_generated_code/raw_code_<timestamp>.json`

**Contains**:

- `raw_response` - The exact text returned by Gemini (before ANY processing)
- `processed_code` - The code after basic cleanup (removing markdown, etc.)
- `prompt` - The prompt sent to Gemini (truncated)
- `model_used` - Which Gemini model was used
- `language` - Target programming language
- `has_markdown` - Whether the response contained markdown formatting

**Example**:

````json
{
  "timestamp": "2026-01-11T11:50:00.000000",
  "language": "javascript",
  "model_used": "gemini:gemini-1.5-flash-latest",
  "raw_response": "```javascript\nfunction hello() {\n  console.log('Hello');\n}\n```",
  "processed_code": "function hello() {\n  console.log('Hello');\n}",
  "code_length": 45,
  "has_markdown": true
}
````

### 2. Validation Results

**Location**: `data/validation_results/validation_<timestamp>.json`

**Contains**:

- `validation_results[]` - Array of all fixes that were validated
  - `is_safe` - Whether the fix passed validation
  - `errors` - List of validation errors (why it was rejected)
  - `warnings` - Non-blocking warnings
  - `content_preview` - First 1000 chars of the code
  - `path` - Target file path
  - `bug_type` - Type of bug being fixed

**Example**:

```json
{
  "timestamp": "2026-01-11T11:50:00",
  "total_fixes": 3,
  "safe_fixes": 1,
  "rejected_fixes": 2,
  "approval_rate": 33.33,
  "validation_results": [
    {
      "fix_number": 1,
      "path": "index.html",
      "is_safe": false,
      "errors": [
        "Dangerous pattern 'document.body.innerHTML = \"\"': ['document.body.innerHTML = \"\"']"
      ],
      "warnings": [],
      "bug_type": "accessibility"
    }
  ]
}
```

## Troubleshooting Code Generation Failures

### Step 1: Check Raw Generated Code

Look at the latest file in `data/raw_generated_code/`:

```bash
ls -lt data/raw_generated_code/ | head -5
cat data/raw_generated_code/raw_code_<latest>.json
```

**Check for**:

- Is `raw_response` empty? → Gemini didn't generate anything
- Does `raw_response` contain an error message? → Gemini API error
- Does `raw_response` look correct but `processed_code` is wrong? → Processing issue

### Step 2: Check Validation Results

Look at the latest file in `data/validation_results/`:

```bash
ls -lt data/validation_results/ | head -5
cat data/validation_results/validation_<latest>.json
```

**Check for**:

- `is_safe: false` → Fix was rejected by validator
- `errors` array → See exact reasons for rejection
- Common rejection reasons:
  - "Unsafe file location" → File path not in allowed directories
  - "Dangerous pattern" → Code contains potentially harmful operations
  - "syntax error" → Code has syntax errors

### Step 3: Common Issues and Solutions

#### Issue: Generated code looks fine but is rejected

**Likely cause**: Validation is too strict

**Solution**:

1. Check the `errors` in validation results
2. If it's a false positive (e.g., safe element removal), you can:
   - Manually apply the fix
   - Adjust validator settings
   - Mark as bug fix to bypass some checks

#### Issue: No raw generated code files

**Likely cause**: Debug mode is disabled or permissions issue

**Solution**:

```bash
# Enable debug mode
export SAVE_RAW_GENERATED_CODE=true

# Check permissions
ls -la data/
mkdir -p data/raw_generated_code
chmod 755 data/raw_generated_code
```

#### Issue: Empty or error response from Gemini

**Likely cause**: API issue or prompt problem

**Solution**:

1. Check the `model_used` - should be a Gemini model
2. Check the `prompt` - is it too long or malformed?
3. Test Gemini API connection:
   ```bash
   python3 -c "from src.ai_api import test_gemini_connection; test_gemini_connection()"
   ```

## Viewing Debug Files

### Using jq (recommended)

```bash
# View latest raw generated code
jq '.' data/raw_generated_code/$(ls -t data/raw_generated_code/ | head -1)

# View latest validation results
jq '.validation_results[] | select(.is_safe == false)' \
  data/validation_results/$(ls -t data/validation_results/ | head -1)

# See rejection reasons
jq '.validation_results[] | {path, errors}' \
  data/validation_results/$(ls -t data/validation_results/ | head -1)
```

### Using Python

```python
import json
import glob
import os

# Load latest raw generated code
files = sorted(glob.glob('data/raw_generated_code/*.json'), reverse=True)
if files:
    with open(files[0]) as f:
        data = json.load(f)
        print("Raw Response:")
        print(data['raw_response'])
        print("\nProcessed Code:")
        print(data['processed_code'])
```

## Bypassing Validation (For Testing ONLY)

**⚠️ WARNING: This is DANGEROUS and should ONLY be used for debugging!**

If you want to see the code without validation for testing:

1. **Option 1**: Read debug files directly (RECOMMENDED)

   - Raw code is in `data/raw_generated_code/`
   - No need to bypass validation

2. **Option 2**: Temporarily disable validation (NOT RECOMMENDED)
   ```python
   # In generator.py, comment out line 308:
   # safe_fixes = validator.validate_all_fixes(all_fixes)
   # And replace with:
   safe_fixes = all_fixes
   ```

## Example Debugging Session

```bash
# 1. Trigger code generation
# (your normal workflow)

# 2. Check what was generated
echo "=== Latest Raw Code ==="
cat data/raw_generated_code/$(ls -t data/raw_generated_code/ | head -1) | jq '.raw_response'

# 3. Check why it was rejected
echo "=== Validation Errors ==="
cat data/validation_results/$(ls -t data/validation_results/ | head -1) | jq '.validation_results[] | select(.is_safe == false) | .errors'

# 4. View the actual code that was rejected
echo "=== Rejected Code Preview ==="
cat data/validation_results/$(ls -t data/validation_results/ | head -1) | jq '.validation_results[] | select(.is_safe == false) | .content_preview'
```
